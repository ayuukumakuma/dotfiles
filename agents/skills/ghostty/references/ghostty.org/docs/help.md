# Help

How to seek help, a list of common issues and solutions,
and how to report a bug or request a feature.

Ghostty is a personal side project with no commercial backing.
As such, there is no support team or dedicated resources to help you.
The community is very helpful and may provide assistance, and
we try to document as many common issues and solutions as possible.

## Common Issues and Solutions

| Issue | Solution |
| --- | --- |
| Error "missing or unsuitable terminal" when running commands or using SSH. | [Terminfo](https://ghostty.org/docs/help/terminfo.html) |
| macOS window managers such as Yabai or Aerospace act strange | [macOS Tiling Window Managers](https://ghostty.org/docs/help/macos-tiling-wms.html) |
| Ghostty has slow startup or high memory usage on Linux. | [GTK Single Instance](https://ghostty.org/docs/help/gtk-single-instance.html), [`+new-window`](https://ghostty.org/docs/linux/systemd.html) |
| "Unable to acquire OpenGL context" error on Linux | [GTK OpenGL Context](https://ghostty.org/docs/help/gtk-opengl-context.html) |
| macOS defaults to login shells | [macOS Login Shells](https://ghostty.org/docs/help/macos-login-shells.html) |
| CLI tools like Claude Code, Docker, etc. repeatedly flash or flicker | [Synchronized Output](https://ghostty.org/docs/help/synchronized-output.html) |

## Seeking Help

If the common issues and solutions above did not help and you
can't find an answer in the documentation, the following resources
are available:

* [#### GitHub Discussions

  Search for and ask questions from the community on GitHub.](https://github.com/ghostty-org/ghostty/discussions/categories/q-a)
* [#### Discord

  Real-time chat with the community. Community-moderated.](https://discord.gg/ghostty)

## Reporting Bugs or Requesting Features

Please, please, please read the [contributing guide on GitHub](https://github.com/ghostty-org/ghostty/blob/main/CONTRIBUTING.md).

> Important
>
> **GitHub Issues is probably not the right place to go.** This project starts
> all bugs and feature requests as discussions on GitHub Discussions until more
> specific details are known. From there, we convert them into GitHub Issues as
> needed. See the contributing guide for more details.
